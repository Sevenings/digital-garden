# Épico
Um épico é um item de trabalho de alto nível que é **muito grande ou complexo** para ser concluído dentro de um único ciclo de desenvolvimento (geralmente uma _sprint_ de 1 a 4 semanas).

Épicos são usados para agrupar funcionalidades relacionadas ou grandes iniciativas. Eles servem como um "contêiner" para um conjunto de tarefas menores e mais gerenciáveis, geralmente chamadas de "histórias de usuário" (_user stories_).

**Pense no épico como um capítulo de um livro:**

- **O Épico** seria "Implementar o sistema de login social".
- **As Histórias de Usuário** (itens menores no backlog) seriam:
    - "Permitir login via Google";
    - "Permitir login via Facebook";
    - "Adicionar página de redefinição de senha";
    - "Criar testes de integração para login social".

## Relação com [[Estágio/Projetos/12 - Pesquisa GitLab/paginas auxiliares/Backlog\|Backlog]]
A relação é hierárquica:

1. Um épico começa no **topo do _backlog_** (ou como um grupo de itens no backlog).
2. Quando um épico se aproxima de ser trabalhado pela equipe, ele é **decomposto** em histórias de usuário menores e tarefas específicas.
3. Essas tarefas menores são então movidas para a parte superior do _backlog_ do sprint e atribuídas à equipe para desenvolvimento.

Tanto o GitHub quanto o [[Estágio/Projetos/12 - Pesquisa GitLab/Pesquisa Gitlab\|GitLab]] oferecem ferramentas para gerenciar épicos e [[Estágio/Projetos/12 - Pesquisa GitLab/paginas auxiliares/Backlog\|backlogs]]. O GitLab, em particular, tem uma implementação robusta de épicos que abrange vários projetos e subgrupos dentro de sua estrutura unificada, facilitando o gerenciamento de iniciativas complexas em toda a organização.
